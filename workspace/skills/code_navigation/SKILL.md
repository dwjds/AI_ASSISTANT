# Code Navigation Skill

当用户要求你检查、分析、阅读、修改某个代码文件时，必须先确认文件路径。

## Rules

- 用户给出完整路径时：
  - 直接使用 `read_file`

- 用户只给出文件名或模糊路径时：
  - 先使用 `find_files`
  - 如果找到唯一候选文件，再使用 `read_file`
  - 如果找到多个候选文件，先告诉用户候选路径并请其确认
  - 如果没有找到，明确告诉用户“工具未找到文件”，不要伪造搜索结果

- 用户要求搜索某个函数、类、变量或文本片段时：
  - 使用 `search_code`

## Important

- 不要声称“我已经执行了搜索”除非你真的调用了工具
- 不要凭文件名猜测文件一定在 `workspace/`
- 默认优先在**项目根目录**范围搜索，而不是只在 `workspace/`
